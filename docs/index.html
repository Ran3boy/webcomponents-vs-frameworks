ВКР / экспериментальный стенд

Сравнительный анализ Web Components и современных фронтенд-фреймворков

Сайт показывает не только итоговую матрицу, а цепочку доказательств: код реализаций, единые сценарии, raw JSON, processed CSV/JSON, диаграммы и обоснование баллов K1-K10.

Цель и методика

Проверить, как шкала 0-3 из раздела 3.1 применяется на практике к одному UI-компоненту «Каталог элементов».

КомпонентКаталог карточек с поиском, фильтрацией, сортировкой и сбросом.
ТехнологииWeb Components, React, Vue.js, Svelte, Lit.
СценарииРендеринг, обновление состояния, CSS-конфликты, переносимость, tooling.

Демо сценария

Этот блок демонстрирует поведение компонента. Сравниваемые реализации находятся в `apps/`.

Статус данных

Проверяется статус processed данных.

Технические метрики

Таблица строится из `results/processed/technical-metrics.json`.

Диаграммы

SVG-файлы генерируются скриптом `npm run generate-charts` по processed данным.

Матрица K1-K10

Оценки не суммируются в общий рейтинг: они показывают профиль технологии.

Обоснование баллов

Текст загружается из `results/processed/scoring-justification.md` через синхронизированный модуль данных.

Выводы по проектным профилям

        

        

          
Изолируемые виджетыWeb Components и Lit рациональны, когда важны переносимость и защита от внешних CSS-конфликтов.

          
Прикладные SPAReact, Vue.js и Svelte удобны при активной работе с состоянием, tooling и UI-экосистемой.

          
Гибридный подходФреймворки могут собирать приложение, а Web Components/Lit - переносимые элементы дизайн-системы.
