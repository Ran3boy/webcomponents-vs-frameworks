# Сравнительный анализ Web Components и фронтенд-фреймворков

Экспериментальный стенд для ВКР на тему: «Сравнительный анализ технологии Web Components и современных фронтенд-фреймворков для разработки пользовательских интерфейсов».

Проект предназначен для воспроизводимого сравнения нескольких подходов к разработке UI-компонентов: Web Components, Lit, Svelte, Vue.js и React. Все реализации выполняют один и тот же сценарий работы с каталогом элементов и проверяются одинаковыми тестами.

## Состав эксперимента

| Технология | Реализация | Назначение в сравнении |
|---|---|---|
| Web Components | нативный Custom Element + Shadow DOM | проверка стандартной компонентной модели браузера |
| Lit | LitElement поверх Web Components | проверка легковесной надстройки над веб-стандартами |
| Svelte | компилируемый компонент | проверка подхода без Virtual DOM в runtime |
| Vue.js | Vue 3 компонент | проверка реактивного фреймворка |
| React | React компонент | проверка компонентной модели с Virtual DOM |

Во всех реализациях используется единый набор данных `data/dataset-500.json` и одинаковые пользовательские действия: поиск, фильтрация, сортировка и сброс.

## Структура проекта

```text
apps/                 реализации каталога на разных технологиях
  web-components/     нативные Web Components
  lit/                Lit
  svelte/             Svelte
  vue/                Vue.js
  react/              React

data/                 единые датасеты для эксперимента
tests/                скрипты сборки, измерений и проверки
results/              результаты последнего запуска эксперимента
  raw/                необработанные JSON-результаты
  processed/          сводные CSV/JSON-таблицы
  charts/             SVG-диаграммы

docs/                 статический сайт для GitHub Pages
experiment-docs/      описание методики и сценариев проверки
Promts/               служебные промпты для подготовки проекта
```

## Требования

- Node.js 18 или выше.
- npm.
- Chromium устанавливается и используется через Playwright.

## Установка

```bash
npm install
```

## Запуск полного эксперимента

```bash
npm run experiment
```

Команда выполняет полный цикл:

1. генерирует датасеты;
2. собирает все реализации через Vite;
3. измеряет производительность через Playwright;
4. проверяет изоляцию стилей;
5. фиксирует признаки переносимости;
6. собирает итоговые CSV/JSON;
7. генерирует диаграммы;
8. обновляет данные для сайта в `docs/`.

## Отдельные команды

## Проверяемые параметры

| Параметр | Что показывает |
|---|---|
| Размер сборки | объем клиентского JavaScript/CSS после production-сборки |
| FCP | время первой отрисовки контента |
| Начальный рендеринг | время появления каталога после загрузки страницы |
| Обновление при поиске | скорость реакции интерфейса на изменение данных |
| Изоляция стилей | устойчивость компонента к внешним CSS-конфликтам |
| Переносимость | возможность использовать компонент вне исходного фреймворка |

| Технология | Размер сборки, КБ | FCP, мс | Начальный рендеринг, мс | Поиск, мс | Изоляция стилей | Переносимость |

## Вывод по эксперименту

По скорости первичного рендеринга и обновления интерфейса все реализации показали близкие результаты. Наиболее заметное различие зафиксировано в размере клиентской сборки и в изоляции стилей.

Web Components и Lit показали меньший размер сборки и устойчивую изоляцию стилей за счет Shadow DOM. React, Vue.js и Svelte удобнее для разработки приложений внутри собственной экосистемы, но их компоненты сильнее связаны с соответствующим runtime или сборочным процессом.

Практический вывод для ВКР: Web Components рациональны для дизайн-систем, независимых виджетов и долгоживущих компонентов, а фронтенд-фреймворки — для полноценных SPA с маршрутизацией, состоянием и развитой инфраструктурой разработки.
